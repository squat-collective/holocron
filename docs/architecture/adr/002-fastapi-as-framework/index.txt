14:T1d9a,<h1 id="adr-002-fastapi-as-web-framework">ADR-002: FastAPI as Web Framework<a class="anchor" aria-label="Link to section" href="#adr-002-fastapi-as-web-framework">#</a></h1>
<p><strong>Date:</strong> 2026-01-27
<strong>Status:</strong> Accepted
<strong>Deciders:</strong> Tom</p>
<h2 id="context">Context<a class="anchor" aria-label="Link to section" href="#context">#</a></h2>
<p>Holocron needs a REST API as its core interface. The API will be consumed by:</p>
<ul>
<li>Web UI (primary, future)</li>
<li>CLI tools (future)</li>
<li>MCP integrations (future)</li>
<li>Direct API calls</li>
</ul>
<p>Requirements:</p>
<ul>
<li>RESTful endpoints with JSON</li>
<li>Request/response validation</li>
<li>Auto-generated API documentation</li>
<li>Async support (for Neo4j driver)</li>
<li>Type safety</li>
</ul>
<h2 id="decision">Decision<a class="anchor" aria-label="Link to section" href="#decision">#</a></h2>
<p><strong>Use FastAPI as the web framework.</strong></p>
<h2 id="options-considered">Options Considered<a class="anchor" aria-label="Link to section" href="#options-considered">#</a></h2>
<h3 id="1-flask">1. Flask<a class="anchor" aria-label="Link to section" href="#1-flask">#</a></h3>
<p><strong>Pros:</strong></p>
<ul>
<li>Simple, minimal</li>
<li>Large ecosystem</li>
<li>Very flexible</li>
</ul>
<p><strong>Cons:</strong></p>
<ul>
<li>No built-in validation</li>
<li>No auto-generated docs</li>
<li>Sync by default (async is bolt-on)</li>
<li>Manual OpenAPI spec</li>
</ul>
<h3 id="2-django--drf">2. Django + DRF<a class="anchor" aria-label="Link to section" href="#2-django--drf">#</a></h3>
<p><strong>Pros:</strong></p>
<ul>
<li>Batteries included</li>
<li>Strong ORM</li>
<li>Admin interface</li>
</ul>
<p><strong>Cons:</strong></p>
<ul>
<li>Heavy for our use case (no SQL ORM needed)</li>
<li>Opinionated structure</li>
<li>Async support is recent/limited</li>
</ul>
<h3 id="3-fastapi--selected">3. FastAPI ✅ Selected<a class="anchor" aria-label="Link to section" href="#3-fastapi--selected">#</a></h3>
<p><strong>Pros:</strong></p>
<ul>
<li>Native async/await</li>
<li>Pydantic integration (validation + serialization)</li>
<li>Auto-generated OpenAPI docs</li>
<li>Type hints everywhere</li>
<li>Modern Python (3.10+)</li>
</ul>
<p><strong>Cons:</strong></p>
<ul>
<li>Smaller ecosystem than Flask/Django</li>
<li>Less "magic" (more explicit code)</li>
</ul>
<h3 id="4-litestar-formerly-starlite">4. Litestar (formerly Starlite)<a class="anchor" aria-label="Link to section" href="#4-litestar-formerly-starlite">#</a></h3>
<p><strong>Pros:</strong></p>
<ul>
<li>Similar to FastAPI</li>
<li>Some performance improvements</li>
</ul>
<p><strong>Cons:</strong></p>
<ul>
<li>Smaller community</li>
<li>Less documentation</li>
<li>Unnecessary risk for MVP</li>
</ul>
<h2 id="rationale">Rationale<a class="anchor" aria-label="Link to section" href="#rationale">#</a></h2>
<ol>
<li>
<p><strong>Pydantic Integration</strong>: We're already using Pydantic for domain models. FastAPI's native integration means request validation, response serialization, and OpenAPI generation come free.</p>
</li>
<li>
<p><strong>Async Native</strong>: Neo4j's Python driver supports async. FastAPI makes async endpoints natural, not bolted-on.</p>
</li>
<li>
<p><strong>API Documentation</strong>: Auto-generated Swagger/OpenAPI docs help us iterate quickly and help future UI/CLI developers understand the API.</p>
</li>
<li>
<p><strong>Type Safety</strong>: FastAPI enforces type hints, aligning with our "strict typing" guideline.</p>
</li>
</ol>

<h2 id="consequences">Consequences<a class="anchor" aria-label="Link to section" href="#consequences">#</a></h2>
<h3 id="positive">Positive<a class="anchor" aria-label="Link to section" href="#positive">#</a></h3>
<ul>
<li>Instant API documentation at <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>/docs</span></span></code></span></li>
<li>Type errors caught at development time</li>
<li>Clean async code with Neo4j</li>
<li>Pydantic schemas shared between API and domain</li>
</ul>
<h3 id="negative">Negative<a class="anchor" aria-label="Link to section" href="#negative">#</a></h3>
<ul>
<li>Team needs FastAPI knowledge</li>
<li>Some Flask/Django patterns don't apply</li>
<li>Dependency injection style may feel different</li>
</ul>
<h3 id="mitigations">Mitigations<a class="anchor" aria-label="Link to section" href="#mitigations">#</a></h3>
<ul>
<li>FastAPI docs are excellent</li>
<li>Patterns are well-established</li>
<li>Most Python developers pick it up quickly</li>
</ul>
