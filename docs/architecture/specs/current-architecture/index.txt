<h2 id="data-model">Data model<a class="anchor" aria-label="Link to section" href="#data-model">#</a></h2>
<p>Single Neo4j graph. Entities: <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>Asset</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>Actor</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>Rule</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>Event</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>Webhook</span></span></code></span>, plus derived <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>Container</span></span></code></span> and <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>Field</span></span></code></span> from schema projection. Edges: <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>OWNS</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>USES</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>FEEDS</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>CONTAINS</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>MEMBER_OF</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>APPLIES_TO</span></span></code></span>. Multi-label nodes per <a href="/docs/architecture/adr/004-multi-label-node-model">ADR-004</a>. Full reference in <a href="/docs/concepts"><span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>concepts.md</span></span></code></span></a>.</p>
<p>Constraints + indexes are created at startup (<span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>packages/api/src/holocron/db/init.py</span></span></code></span>):</p>
<ul>
<li>Unique constraints on <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>uid</span></span></code></span> for every primary label.</li>
<li>Vector indexes (cosine, 384-dim) on <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>:Asset.embedding</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>:Actor.embedding</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>:Rule.embedding</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>:Container.embedding</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>:Field.embedding</span></span></code></span>.</li>
<li>Fulltext indexes on <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>name + description</span></span></code></span> for the same set.</li>
</ul>
<h2 id="request-flow-read">Request flow (read)<a class="anchor" aria-label="Link to section" href="#request-flow-read">#</a></h2>
<figure data-rehype-pretty-code-figure=""><pre tabindex="0" data-language="plaintext" data-theme="github-dark-dimmed github-light"><code data-language="plaintext" data-theme="github-dark-dimmed github-light" style="display: grid;"><span data-line=""><span>Browser ──► UI route handler ──► HOLOCRON_API_URL ──► FastAPI route</span></span>
<span data-line=""><span>                                                       ├─ dependency injection (services, settings)</span></span>
<span data-line=""><span>                                                       ├─ rate limiter</span></span>
<span data-line=""><span>                                                       └─ service ─► repository ─► Neo4j (Bolt async)</span></span>
<span data-line=""><span>                                                       ◄────────── response model ◄──────</span></span></code></pre></figure>
<p>The UI never talks to Neo4j directly. The browser never talks to the API directly — it goes through Next.js route handlers under <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>/api/holocron/*</span></span></code></span> so the API base URL never leaks client-side.</p>
<h2 id="request-flow-write">Request flow (write)<a class="anchor" aria-label="Link to section" href="#request-flow-write">#</a></h2>
<p>Same path, plus:</p>
<ol>
<li>The service writes the entity through the repository.</li>
<li>The service updates the embedding (if applicable) via <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>embedding_service.embed_one()</span></span></code></span>.</li>
<li>If the entity is an Asset with <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>metadata.schema</span></span></code></span>, the schema projection rebuilds the <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>:Container</span></span></code></span> / <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>:Field</span></span></code></span> subtree.</li>
<li>The service logs an <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>:Event</span></span></code></span> via <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>event_service.log()</span></span></code></span>.</li>
<li><span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>event_service</span></span></code></span> triggers <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>webhook_dispatcher.dispatch_event()</span></span></code></span> which fans out HMAC-signed POSTs in <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>asyncio</span></span></code></span> background tasks (bounded by a process-wide semaphore of 100).</li>
<li>The API response returns immediately — webhooks don't block.</li>
</ol>

<h2 id="known-limitations">Known limitations<a class="anchor" aria-label="Link to section" href="#known-limitations">#</a></h2>
<ul>
<li><strong>No auth.</strong> Production deployments must put a proxy in front. See <a href="/docs/deployment#production-notes">deployment.md</a>.</li>
<li><strong>No soft-delete.</strong> Hard deletes only. Recovery path is the event log + re-import.</li>
<li><strong>Schema projection is full rebuild on every asset write.</strong> Bounded by per-asset schema size. At scale, move to a batched/incremental rebuild.</li>
<li><strong>Graph layout cached at startup.</strong> Adding many entities later doesn't re-layout — restart to refresh.</li>
<li><strong>No persistent webhook retry.</strong> A failed delivery is not retried beyond the immediate attempt; 10 failures auto-disable.</li>
<li><strong>Webhook secrets stored plaintext.</strong> The dispatcher needs them to sign; treat the DB as sensitive.</li>
<li><strong>Embedding model loads lazily.</strong> First search after start is slower (~1 s). Pre-warm if needed.</li>
<li><strong>Search DSL is forgiving by design.</strong> Unknown operators silently become semantic queries.</li>
