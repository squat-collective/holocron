14:Ta24e,<h1 id="plugins">Plugins<a class="anchor" aria-label="Link to section" href="#plugins">#</a></h1>
<blockquote>
<p>Connectors push, exporters pull, audits report. All of it runs through the same plugin contract — auto-discovered at API startup, driven by manifest, invoked over HTTP.</p>
</blockquote>
<h2 id="how-plugins-work">How plugins work<a class="anchor" aria-label="Link to section" href="#how-plugins-work">#</a></h2>
<ol>
<li>A plugin is a Python package that exposes two attributes from one module:
<ul>
<li><span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>manifest: PluginManifest</span></span></code></span> — slug, name, capability, inputs, etc.</li>
<li><span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>run: async (ctx: PluginContext, inputs: dict) -> SummaryResult | DownloadResult</span></span></code></span></li>
</ul>
</li>
<li>The package registers itself via the <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>holocron.plugins</span></span></code></span> entry point in <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>pyproject.toml</span></span></code></span>.</li>
<li>The API discovers the entry-point group at startup (<span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>PluginRegistry.discover()</span></span></code></span>), loads each module, and exposes them at:
<ul>
<li><span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>GET /api/v1/plugins</span></span></code></span> — list of manifests.</li>
<li><span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>POST /api/v1/plugins/{slug}/run</span></span></code></span> — multipart form invocation.</li>
</ul>
</li>
<li>Inside <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>run()</span></span></code></span>, the plugin gets a <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>PluginContext</span></span></code></span> carrying the API's service layer (<span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>asset_service</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>actor_service</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>relation_service</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>rule_service</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>event_service</span></span></code></span>) — so it can read and mutate the catalog without going through HTTP.</li>
<li>The result is either a <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>SummaryResult</span></span></code></span> (returned as JSON) or a <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>DownloadResult</span></span></code></span> (streamed with <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>Content-Disposition: attachment</span></span></code></span>).</li>
</ol>
<p>There is <strong>one</strong> plugin contract and it lives in the <a href="../packages/holocron-plugin-sdk/"><span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>holocron-plugin-sdk</span></span></code></span></a> package — both the API and every plugin import from there.</p>
<p>See <a href="/docs/architecture/adr/006-plugin-sdk-entry-points">ADR-006</a> for the design choice; <a href="/docs/architecture/adr/003-reader-plugin-architecture">ADR-003</a> records the original "reader" design that this supersedes.</p>

<h3 id="conventions">Conventions<a class="anchor" aria-label="Link to section" href="#conventions">#</a></h3>
<ul>
<li>Set <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>verified=False</span></span></code></span> and <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>discovered_by="&#x3C;slug>@&#x3C;version>"</span></span></code></span> on anything you create — it's the audit trail.</li>
<li>Use <strong>deterministic UIDs</strong> (hash of source identity) so re-runs upsert instead of duplicating.</li>
<li>Keep <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>manifest.icon</span></span></code></span> to a single emoji — that's what shows on the UI plugin card.</li>
<li>Don't import from <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>holocron.*</span></span></code></span> — only from <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>holocron_plugin_sdk</span></span></code></span>. The SDK is the public boundary.</li>
<li>mypy strict, pydantic 2.10+. Same conventions as the rest of the monorepo.</li>
</ul>

<h2 id="operational-concerns">Operational concerns<a class="anchor" aria-label="Link to section" href="#operational-concerns">#</a></h2>
<ul>
<li><strong>Plugins share the API process.</strong> A misbehaving plugin can affect API performance. Treat plugins like first-party code.</li>
<li><strong>Discovery is at startup.</strong> Adding or removing a plugin needs an API restart.</li>
<li><strong>Errors propagate.</strong> A <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>ValueError</span></span></code></span> from <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>run()</span></span></code></span> becomes a 422 with the message; everything else becomes a 500. Use <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>ValueError</span></span></code></span> for user-facing input problems.</li>
<li><strong>No persistent job queue.</strong> Plugin runs are synchronous from the API's point of view (the request is held open until the plugin returns). Long-running scans are fine but consume an API worker.</li>
