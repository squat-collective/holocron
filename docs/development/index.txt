14:T665e,<h1 id="development">Development<a class="anchor" aria-label="Link to section" href="#development">#</a></h1>
<blockquote>
<p>How to work on Holocron without installing anything on your host.</p>
</blockquote>
<h2 id="ground-rules">Ground rules<a class="anchor" aria-label="Link to section" href="#ground-rules">#</a></h2>
<ul>
<li><strong>Containerised only.</strong> Podman first, Docker as fallback. The <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>Makefile</span></span></code></span> auto-detects whichever is on <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>$PATH</span></span></code></span>.</li>
<li><strong>TDD.</strong> Write the failing test, then the code. Every package is configured to run tests in a container with one command.</li>
<li><strong>KISS.</strong> Build what's needed. No speculative abstractions. No half-finished features.</li>
<li><strong>Strict types.</strong> mypy <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>--strict</span></span></code></span> for Python, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>strict: true</span></span></code></span> for TypeScript, no <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>any</span></span></code></span>.</li>
<li><strong>One PR updates docs and code together.</strong> If you change behaviour, update the doc that describes it.</li>
</ul>
<p>Each package's <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>CLAUDE.md</span></span></code></span> has the package-specific take on these rules.</p>

<h3 id="dev-image-vs-released-image">Dev image vs released image<a class="anchor" aria-label="Link to section" href="#dev-image-vs-released-image">#</a></h3>
<p>The dev compose builds <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>packages/api/Dockerfile</span></span></code></span> with <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>INSTALL_EXTRAS=[dev]</span></span></code></span> so pytest, mypy, and ruff land in the API container — that's why <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>make api-test</span></span></code></span> works. The release workflow builds the same Dockerfile <strong>without</strong> the arg, producing a slimmer (~94 MB lighter) GHCR image with runtime deps only. If you need to mirror the prod image locally:</p>
<figure data-rehype-pretty-code-figure=""><pre tabindex="0" data-language="bash" data-theme="github-dark-dimmed github-light"><code data-language="bash" data-theme="github-dark-dimmed github-light" style="display: grid;"><span data-line=""><span style="--shiki-dark:#F69D50;--shiki-light:#6F42C1">podman</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> build</span><span style="--shiki-dark:#6CB6FF;--shiki-light:#005CC5"> -f</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> packages/api/Dockerfile</span><span style="--shiki-dark:#6CB6FF;--shiki-light:#005CC5"> -t</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> holocron-api:slim</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> .</span></span></code></pre></figure>
<p>The UI image is multi-stage (<span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>packages/ui/Dockerfile</span></span></code></span>) and only used for releases — local dev runs Next as <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>bun dev</span></span></code></span> against bind-mounted source.</p>
<h2 id="adding-a-feature-full-loop">Adding a feature: full loop<a class="anchor" aria-label="Link to section" href="#adding-a-feature-full-loop">#</a></h2>
<p>The "right" loop is the same regardless of which package you touch:</p>
<ol>
<li><strong>Write the failing test.</strong> Pytest for API, Vitest for UI, Bun test for SDK / MCP.</li>
<li><strong>Implement until green.</strong></li>
<li><strong>Lint + typecheck.</strong> <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>make api-lint api-typecheck</span></span></code></span> (Python) or <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>make ui-typecheck</span></span></code></span> / <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>make sdk-typecheck</span></span></code></span> (TS).</li>
<li><strong>Run the full suite.</strong> <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>make api-test</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>make ui-test</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>make sdk-test</span></span></code></span>.</li>
<li><strong>If you touched the API surface, regenerate docs.</strong> <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>make openapi</span></span></code></span>. The SDK regenerates its types from the spec via <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>bun run generate-types</span></span></code></span>.</li>
<li><strong>Update the relevant doc page.</strong> Don't ship behaviour changes without updating <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>docs/</span></span></code></span>.</li>
<li><strong>Commit.</strong> Conventional-style messages (<span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>feat(api):</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>fix(ui):</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>refactor(...)</span></span></code></span>, ...). The git history is the changelog.</li>
</ol>
<h2 id="adding-an-api-endpoint">Adding an API endpoint<a class="anchor" aria-label="Link to section" href="#adding-an-api-endpoint">#</a></h2>
<ol>
<li>Pydantic schema in <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>packages/api/src/holocron/api/schemas/</span></span></code></span>.</li>
<li>Service method in <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>packages/api/src/holocron/core/services/</span></span></code></span>.</li>
<li>Repository method in <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>packages/api/src/holocron/db/repositories/</span></span></code></span> if it touches Neo4j.</li>
<li>Route in <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>packages/api/src/holocron/api/routes/</span></span></code></span> — call the service, raise the right exceptions.</li>
<li>Test (unit for the service, integration for the route).</li>
<li><span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>make openapi</span></span></code></span> to refresh <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>docs/openapi.json</span></span></code></span>.</li>
<li>Add a method to the SDK if it's a public API.</li>
</ol>
<p>Patterns to copy: <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>routes/assets.py</span></span></code></span> is the canonical CRUD, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>routes/webhooks.py</span></span></code></span> is the canonical "create returns extra-once" pattern.</p>
<h2 id="adding-a-ui-feature">Adding a UI feature<a class="anchor" aria-label="Link to section" href="#adding-a-ui-feature">#</a></h2>
<ol>
<li>Test first (Vitest + RTL).</li>
<li>If the feature needs API data, check the SDK has a method. If not, <strong>add it to the SDK first</strong>, rebuild (<span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>make sdk-build</span></span></code></span>), then consume it from the UI.</li>
<li>Server components by default; <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>'use client'</span></span></code></span> only when you need state, effects, or browser APIs.</li>
<li>Data fetching: TanStack Query in client components, direct SDK call in server components / route handlers.</li>
<li>Style with Tailwind v4 + shadcn/ui primitives. The galaxy palette is in <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>globals.css</span></span></code></span> as oklch tokens.</li>
<li>If it's a command, expose it via an <a href="/docs/extensions">extension</a> so it shows up in ⌘K.</li>
</ol>
