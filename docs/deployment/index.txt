14:T84ac,<h1 id="deployment">Deployment<a class="anchor" aria-label="Link to section" href="#deployment">#</a></h1>
<blockquote>
<p>Holocron ships as pre-built images on GHCR. One command gets you a running stack — <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>curl … | bash</span></span></code></span> and you're done.</p>
</blockquote>
<p>There are three deployment shapes, in increasing order of effort and exposure:</p>
<ol>
<li><strong>One-click installer</strong> — pulls images from GHCR, generates a password, runs compose. LAN-only.</li>
<li><strong>Caddy overlay</strong> — same as above, fronted by Caddy with optional basic auth and auto-HTTPS. Internet-facing.</li>
<li><strong>Build from source</strong> — the original dev compose at the repo root. For contributors only.</li>
</ol>
<h2 id="1-one-click-install">1. One-click install<a class="anchor" aria-label="Link to section" href="#1-one-click-install">#</a></h2>
<figure data-rehype-pretty-code-figure=""><pre tabindex="0" data-language="bash" data-theme="github-dark-dimmed github-light"><code data-language="bash" data-theme="github-dark-dimmed github-light" style="display: grid;"><span data-line=""><span style="--shiki-dark:#F69D50;--shiki-light:#6F42C1">curl</span><span style="--shiki-dark:#6CB6FF;--shiki-light:#005CC5"> -fsSL</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> https://github.com/squat-collective/holocron/releases/latest/download/install.sh</span><span style="--shiki-dark:#F47067;--shiki-light:#D73A49"> |</span><span style="--shiki-dark:#F69D50;--shiki-light:#6F42C1"> bash</span></span></code></pre></figure>
<p>That's it. The installer:</p>
<ul>
<li>Detects <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>docker</span></span></code></span> or <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>podman</span></span></code></span> (in that order) and refuses to continue without one.</li>
<li>Creates <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>./holocron/</span></span></code></span> with <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>compose.prod.yml</span></span></code></span> and <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>.env</span></span></code></span>.</li>
<li>Generates a strong <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>NEO4J_PASSWORD</span></span></code></span> (32 chars) on first run.</li>
<li><span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>compose pull &#x26;&#x26; compose up -d</span></span></code></span>.</li>
<li>Polls <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>GET /api/v1/health</span></span></code></span> until the API is up, then prints the URLs.</li>
</ul>
<p><strong>Pin to a release</strong> instead of <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>latest</span></span></code></span>:</p>
<figure data-rehype-pretty-code-figure=""><pre tabindex="0" data-language="bash" data-theme="github-dark-dimmed github-light"><code data-language="bash" data-theme="github-dark-dimmed github-light" style="display: grid;"><span data-line=""><span style="--shiki-dark:#F69D50;--shiki-light:#6F42C1">curl</span><span style="--shiki-dark:#6CB6FF;--shiki-light:#005CC5"> -fsSL</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> https://github.com/squat-collective/holocron/releases/download/v0.1.0/install.sh</span><span style="--shiki-dark:#F47067;--shiki-light:#005CC5"> \</span></span>
<span data-line=""><span style="--shiki-dark:#F47067;--shiki-light:#D73A49">  |</span><span style="--shiki-dark:#ADBAC7;--shiki-light:#24292E"> HOLOCRON_VERSION</span><span style="--shiki-dark:#F47067;--shiki-light:#D73A49">=</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62">v0.1.0</span><span style="--shiki-dark:#F69D50;--shiki-light:#6F42C1"> bash</span></span></code></pre></figure>
<p><strong>Custom install dir:</strong></p>
<figure data-rehype-pretty-code-figure=""><pre tabindex="0" data-language="bash" data-theme="github-dark-dimmed github-light"><code data-language="bash" data-theme="github-dark-dimmed github-light" style="display: grid;"><span data-line=""><span style="--shiki-dark:#F69D50;--shiki-light:#6F42C1">curl</span><span style="--shiki-dark:#6CB6FF;--shiki-light:#005CC5"> -fsSL</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> .../install.sh</span><span style="--shiki-dark:#F47067;--shiki-light:#D73A49"> |</span><span style="--shiki-dark:#ADBAC7;--shiki-light:#24292E"> HOLOCRON_DIR</span><span style="--shiki-dark:#F47067;--shiki-light:#D73A49">=</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62">/opt/holocron</span><span style="--shiki-dark:#F69D50;--shiki-light:#6F42C1"> bash</span></span></code></pre></figure>
<p>Re-running the installer is safe: it won't overwrite an existing <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>.env</span></span></code></span>. To upgrade, change <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>HOLOCRON_VERSION</span></span></code></span> in <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>.env</span></span></code></span> and run <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>make prod-pull &#x26;&#x26; make prod-up</span></span></code></span>.</p>

<h2 id="2-caddy-overlay-public-facing">2. Caddy overlay (public-facing)<a class="anchor" aria-label="Link to section" href="#2-caddy-overlay-public-facing">#</a></h2>
<p>The bare prod compose has no auth. To put it on the open internet, layer the Caddy overlay on top:</p>
<figure data-rehype-pretty-code-figure=""><pre tabindex="0" data-language="bash" data-theme="github-dark-dimmed github-light"><code data-language="bash" data-theme="github-dark-dimmed github-light" style="display: grid;"><span data-line=""><span style="--shiki-dark:#F69D50;--shiki-light:#6F42C1">cp</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> Caddyfile.example</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> Caddyfile</span></span>
<span data-line=""><span style="--shiki-dark:#768390;--shiki-light:#6A737D"># edit Caddyfile: set domain, optionally enable basic_auth</span></span>
<span data-line=""><span style="--shiki-dark:#6CB6FF;--shiki-light:#005CC5">echo</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> 'HOLOCRON_DOMAIN=holocron.example.com'</span><span style="--shiki-dark:#F47067;--shiki-light:#D73A49"> >></span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> .env</span></span>
<span data-line=""><span style="--shiki-dark:#F69D50;--shiki-light:#6F42C1">make</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> prod-up-caddy</span></span></code></pre></figure>
<p>The overlay (<span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>compose.prod.caddy.yml</span></span></code></span>) adds a Caddy container that:</p>
<ul>
<li>Routes <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>/api/*</span></span></code></span> to the API container, everything else to the UI.</li>
<li>Auto-issues a Let's Encrypt cert when <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>HOLOCRON_DOMAIN</span></span></code></span> is a real hostname.</li>
<li>Optionally basic-auths every request (uncomment the block in <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>Caddyfile</span></span></code></span>).</li>
</ul>
<blockquote>
<p><strong>Important:</strong> the base <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>compose.prod.yml</span></span></code></span> still binds host ports for the UI, API, and Neo4j. To make Caddy the only public entry point, override those <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>ports:</span></span></code></span> to bind <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>127.0.0.1</span></span></code></span> in your own <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>compose.override.yml</span></span></code></span>.</p>
</blockquote>
<h2 id="3-build-from-source">3. Build from source<a class="anchor" aria-label="Link to section" href="#3-build-from-source">#</a></h2>

<h2 id="production-hardening-checklist">Production hardening checklist<a class="anchor" aria-label="Link to section" href="#production-hardening-checklist">#</a></h2>
<p>The one-click install is "good enough" for an internal LAN. Before exposing it publicly:</p>
<ul>
<li>✅ Pin <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>HOLOCRON_VERSION</span></span></code></span> to a real tag (not <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>latest</span></span></code></span>).</li>
<li>✅ Use the Caddy overlay or your own reverse proxy. The API and UI have no auth.</li>
<li>✅ Set a strong <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>NEO4J_PASSWORD</span></span></code></span> (the installer does this for you).</li>
<li>⚠️ Pin upstream image digests too (<span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>neo4j:5</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>caddy:2</span></span></code></span>) if you need byte-identical deploys.</li>
<li>⚠️ Hit any search endpoint after deploy to warm the embedding model into RAM (~300 MB). Pre-warm at boot if first-request latency matters.</li>
<li>⚠️ For high throughput, run multiple API workers behind the proxy. The webhook dispatcher's semaphore is per-process; more workers = more concurrent dispatches.</li>
<li>⚠️ Set up backups for <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>neo4j_data</span></span></code></span>.</li>
</ul>
