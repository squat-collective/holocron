14:T5e89,<h1 id="getting-started">Getting started<a class="anchor" aria-label="Link to section" href="#getting-started">#</a></h1>
<blockquote>
<p>Bring up the stack, create your first asset, import a CSV, run an audit, and see results in the UI — in about 10 minutes.</p>
</blockquote>
<h2 id="prerequisites">Prerequisites<a class="anchor" aria-label="Link to section" href="#prerequisites">#</a></h2>
<ul>
<li><strong>Podman</strong> or <strong>Docker</strong> with <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>compose</span></span></code></span> support.</li>
<li>That's it. Nothing else needs to be installed on the host.</li>
</ul>
<h2 id="1-start-the-stack">1. Start the stack<a class="anchor" aria-label="Link to section" href="#1-start-the-stack">#</a></h2>
<p>There are two paths depending on what you're doing.</p>
<h3 id="a-run-holocron-one-click-install">a) Run Holocron (one-click install)<a class="anchor" aria-label="Link to section" href="#a-run-holocron-one-click-install">#</a></h3>
<figure data-rehype-pretty-code-figure=""><pre tabindex="0" data-language="bash" data-theme="github-dark-dimmed github-light"><code data-language="bash" data-theme="github-dark-dimmed github-light" style="display: grid;"><span data-line=""><span style="--shiki-dark:#F69D50;--shiki-light:#6F42C1">curl</span><span style="--shiki-dark:#6CB6FF;--shiki-light:#005CC5"> -fsSL</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> https://github.com/squat-collective/holocron/releases/latest/download/install.sh</span><span style="--shiki-dark:#F47067;--shiki-light:#D73A49"> |</span><span style="--shiki-dark:#F69D50;--shiki-light:#6F42C1"> bash</span></span></code></pre></figure>
<p>Pulls pre-built images from GHCR, generates a strong Neo4j password into <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>./holocron/.env</span></span></code></span>, brings everything up, and waits until <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>/health</span></span></code></span> is green. This is the recommended path for users.</p>
<p>To pin a release: <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>… | HOLOCRON_VERSION=v0.1.0 bash</span></span></code></span>. To upgrade later: <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>… | bash -s -- --update</span></span></code></span> (see <a href="/docs/deployment#upgrading-to-a-new-release">Upgrading</a>). To put it on the open internet, see the <a href="/docs/deployment#2-caddy-overlay-public-facing">Caddy overlay</a>.</p>
<h3 id="b-develop-on-holocron-build-from-source">b) Develop on Holocron (build from source)<a class="anchor" aria-label="Link to section" href="#b-develop-on-holocron-build-from-source">#</a></h3>
<figure data-rehype-pretty-code-figure=""><pre tabindex="0" data-language="bash" data-theme="github-dark-dimmed github-light"><code data-language="bash" data-theme="github-dark-dimmed github-light" style="display: grid;"><span data-line=""><span style="--shiki-dark:#F69D50;--shiki-light:#6F42C1">git</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> clone</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> https://github.com/squat-collective/holocron.git</span><span style="--shiki-dark:#ADBAC7;--shiki-light:#24292E"> &#x26;&#x26; </span><span style="--shiki-dark:#6CB6FF;--shiki-light:#005CC5">cd</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> holocron</span></span>
<span data-line=""><span style="--shiki-dark:#F69D50;--shiki-light:#6F42C1">make</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> up</span><span style="--shiki-dark:#768390;--shiki-light:#6A737D">         # builds the API locally with dev extras, runs UI as `bun dev`</span></span>
<span data-line=""><span style="--shiki-dark:#F69D50;--shiki-light:#6F42C1">make</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> health</span></span></code></pre></figure>
<p>The dev compose mounts source for live reload and includes pytest/mypy/ruff in the API container. Use this only if you're hacking on Holocron itself.</p>

<h2 id="2-open-the-ui">2. Open the UI<a class="anchor" aria-label="Link to section" href="#2-open-the-ui">#</a></h2>
<p>Visit <a href="http://localhost:3333">http://localhost:3333</a>. You'll land on the search shell with a galaxy nebula in the background.</p>
<p>There's nothing in the catalog yet, so the search returns no hits. Click the <strong>mode toggle</strong> in the top right to switch to the <strong>galaxy map</strong> — also empty — then back.</p>
<h2 id="3-create-your-first-asset-ui">3. Create your first asset (UI)<a class="anchor" aria-label="Link to section" href="#3-create-your-first-asset-ui">#</a></h2>
<p>Hit <strong>⌘K</strong> (or <strong>Ctrl+K</strong>) to open the command palette. Type <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>create dataset</span></span></code></span>. Pick the wizard, fill in:</p>
<ul>
<li><strong>Name:</strong> <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>customers</span></span></code></span></li>
<li><strong>Type:</strong> <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>dataset</span></span></code></span></li>
<li><strong>Description:</strong> <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>Customer master table</span></span></code></span></li>
</ul>
<p>Hit <strong>Enter</strong> through the steps, <strong>Enter</strong> to confirm. The asset appears in search.</p>

<h2 id="5-import-a-csv">5. Import a CSV<a class="anchor" aria-label="Link to section" href="#5-import-a-csv">#</a></h2>
<p>The <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>csv-connector</span></span></code></span> plugin is bundled. Use the CLI:</p>
<figure data-rehype-pretty-code-figure=""><pre tabindex="0" data-language="bash" data-theme="github-dark-dimmed github-light"><code data-language="bash" data-theme="github-dark-dimmed github-light" style="display: grid;"><span data-line=""><span style="--shiki-dark:#768390;--shiki-light:#6A737D"># Install the SDK CLI inside the API container (or any Python env)</span></span>

<p>Or, in the UI: ⌘K → <strong>Import</strong> → <strong>CSV connector</strong> → upload a file → run.</p>
<p>The plugin scans the file, infers column types, parses any <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span># Owner:</span></span></code></span> comment headers as Person actors, and creates one <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>dataset</span></span></code></span> asset with the schema projected as Container/Field nodes. The result lands <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>verified: false</span></span></code></span> so you can review before trusting it.</p>
<h2 id="6-search">6. Search<a class="anchor" aria-label="Link to section" href="#6-search">#</a></h2>
<p>Back in the UI search box:</p>
<figure data-rehype-pretty-code-figure=""><pre tabindex="0" data-language="plaintext" data-theme="github-dark-dimmed github-light"><code data-language="plaintext" data-theme="github-dark-dimmed github-light" style="display: grid;"><span data-line=""><span>customers                 # semantic + keyword across everything</span></span>
<span data-line=""><span>ds:customers              # only datasets</span></span>
<span data-line=""><span>"customer master"         # literal phrase</span></span>
<span data-line=""><span>owner:Tom                 # owned by anyone matching "Tom"</span></span>
<span data-line=""><span>ds:order -archived        # datasets matching "order", excluding "archived"</span></span></code></pre></figure>
<p>Full reference: <a href="/docs/search">search.md</a>.</p>
<h2 id="7-visit-the-galaxy-map">7. Visit the galaxy map<a class="anchor" aria-label="Link to section" href="#7-visit-the-galaxy-map">#</a></h2>
<p>Switch to <strong>map mode</strong> (toggle, top right). You'll see your assets and any auto-created actors as floating nodes connected by edges. Click one to focus; <strong>Shift+Enter</strong> to lock; arrow keys / vim keys to navigate.</p>
<p>Full reference: <a href="/docs/map">map.md</a>.</p>
<h2 id="8-run-an-audit">8. Run an audit<a class="anchor" aria-label="Link to section" href="#8-run-an-audit">#</a></h2>
<figure data-rehype-pretty-code-figure=""><pre tabindex="0" data-language="bash" data-theme="github-dark-dimmed github-light"><code data-language="bash" data-theme="github-dark-dimmed github-light" style="display: grid;"><span data-line=""><span style="--shiki-dark:#F69D50;--shiki-light:#6F42C1">podman</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> exec</span><span style="--shiki-dark:#6CB6FF;--shiki-light:#005CC5"> -it</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> holocron</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> holocron-plugin</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> run</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> lineage-gap-audit</span><span style="--shiki-dark:#F47067;--shiki-light:#005CC5"> \</span></span>
<span data-line=""><span style="--shiki-dark:#6CB6FF;--shiki-light:#005CC5">  --output</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> audit.xlsx</span></span></code></pre></figure>

<h2 id="9-subscribe-to-events-webhooks">9. Subscribe to events (webhooks)<a class="anchor" aria-label="Link to section" href="#9-subscribe-to-events-webhooks">#</a></h2>
<p>Register a webhook to receive everything that happens:</p>
<figure data-rehype-pretty-code-figure=""><pre tabindex="0" data-language="bash" data-theme="github-dark-dimmed github-light"><code data-language="bash" data-theme="github-dark-dimmed github-light" style="display: grid;"><span data-line=""><span style="--shiki-dark:#F69D50;--shiki-light:#6F42C1">curl</span><span style="--shiki-dark:#6CB6FF;--shiki-light:#005CC5"> -X</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> POST</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> http://localhost:8100/api/v1/webhooks</span><span style="--shiki-dark:#F47067;--shiki-light:#005CC5"> \</span></span>
<span data-line=""><span style="--shiki-dark:#6CB6FF;--shiki-light:#005CC5">  -H</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> 'Content-Type: application/json'</span><span style="--shiki-dark:#F47067;--shiki-light:#005CC5"> \</span></span>
<span data-line=""><span style="--shiki-dark:#6CB6FF;--shiki-light:#005CC5">  -d</span><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62"> '{</span></span>
<span data-line=""><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62">    "url": "https://your-receiver.example.com/hook",</span></span>
<span data-line=""><span style="--shiki-dark:#96D0FF;--shiki-light:#032F62">    "events": ["*"]</span></span>

Every dispatch is HMAC-signed; verify on your end with the recipe in <a href="/docs/webhooks">webhooks.md</a>.</p>
