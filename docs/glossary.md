# Glossary

> Every Holocron-specific term in one place.

## Catalog

**Asset**
A piece of data or something that produces/consumes data. Has a `type` of `dataset`, `report`, `process`, or `system`. Carries `:Asset` plus a type-specific label in Neo4j.

**Actor**
A person or a team. Has a `type` of `person` or `group`. Carries `:Actor` + `:Person` or `:Group`.

**Container**
A schema container — typically a table, sheet, or section. Created automatically by [schema projection](#schema-projection); not directly editable.

**Field**
A schema column or measure. Lives under a Container. Created by schema projection.

**Rule**
A governance / data-quality rule. Has `severity` (`info`, `warning`, `critical`) and a `category`. Attaches to assets via the `applies_to` relation, optionally scoped to a `field_path`.

**Event**
An immutable audit record of one mutation (`asset.created`, `relation.deleted`, ...). Drives outbound webhooks.

**Webhook**
An outbound subscription: URL + event topics + HMAC secret. The API POSTs signed payloads when matching events fire.

## Relationships

**`:OWNS`** — `Actor → Asset`. Who is responsible.
**`:USES`** — `Actor | Asset → Asset`. Who consumes.
**`:FEEDS`** — `Asset → Asset`. **Lineage** edge — source flows to target.
**`:CONTAINS`** — `Asset | Container → Container | Field`. Schema tree membership.
**`:MEMBER_OF`** — `Person → Group`. Team membership.
**`:APPLIES_TO`** — `Rule → Asset`. Governance attachment, with `enforcement` and optional `field_path`.

## Search

**Hybrid search**
The combination of vector (semantic) and fulltext (keyword) ranking, fused into one score per result.

**Embedding**
A 384-dim float vector representing an entity's `name + description`. Generated by BAAI/bge-small-en-v1.5 via fastembed (CPU).

**Query DSL**
The small grammar that lets you write `ds:`, `owner:`, `feeds:`, `"phrase"`, `-exclude` instead of plain text. See [search.md](search.md).

**Kind**
The bucket a search hit belongs to: `asset`, `container`, `field`, `actor`, `rule`. Returned in every search hit.

**Schema projection**
The process of materialising an Asset's `metadata.schema` JSON into real `:Container` and `:Field` nodes so they're searchable individually. Rebuilt on every asset write.

## Plugins

**Plugin**
A Python package exposing `manifest` + `async run()`, registered via the `holocron.plugins` entry-point group. Discovered by the API at startup.

**Capability**
`IMPORT` (returns `SummaryResult`) or `EXPORT` (returns `DownloadResult`).

**`PluginContext`**
The services injected into `run()` — `asset_service`, `actor_service`, `relation_service`, `rule_service`, `event_service`. Lets the plugin read/write the catalog without going through HTTP.

**`SummaryResult`**
The return type for IMPORT plugins: title, counts, samples, optional `extra` dict.

**`DownloadResult`**
The return type for EXPORT plugins: filename, content type, body bytes.

**Connector**
A plugin that pulls metadata from an external source (CSV, Excel, Postgres, Power BI, ...) and pushes it into the catalog.

**Exporter**
A plugin that produces a file from the catalog (Excel workbook, Markdown zip, ...).

**Extension (audit)**
A plugin in the audit / analysis category — `lineage-gap-audit`, `compliance-report`, `pii-detector`. (Not to be confused with UI extensions.)

**UI extension**
A piece of code that contributes commands to the ⌘K palette. Lives under `packages/ui/src/extensions/`. See [extensions.md](extensions.md).

**`holocron-plugin` CLI**
The command-line driver bundled with the SDK. Lists / inspects / invokes plugins on a running API.

## Identity & audit

**UID**
The stable identifier assigned to every entity at creation. Strings like `ast-...`, `act-...`, `rul-...`, `evt-...`, `wh-...`. Connectors typically derive UIDs deterministically from source identity so re-scans upsert.

**`verified`**
Boolean flag. `true` means a human has confirmed; `false` means a plugin guessed it. Connectors create things with `verified: false` and `discovered_by: <slug>@<version>`.

**`discovered_by`**
String tag identifying the plugin (and version) that created the entity, so you can audit provenance.

## Operational

**LOD (Level of Detail)**
Controls how much of the graph the map endpoint returns. `lod=0` is overview-only (systems + teams), `lod=1` is everything.

**Topic**
The `<entity>.<action>` string used in webhook subscriptions and `X-Holocron-Topic`. E.g. `asset.updated`.

**Auto-disable**
After 10 consecutive delivery failures, a webhook is automatically disabled. Re-enable by `PUT`-ing `{"disabled": false}`.

**Embedding model warmup**
The first search request after API start loads the embedding model (~1 s). After that, queries are sub-100 ms.
