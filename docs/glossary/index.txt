14:T5270,<h1 id="glossary">Glossary<a class="anchor" aria-label="Link to section" href="#glossary">#</a></h1>
<blockquote>
<p>Every Holocron-specific term in one place.</p>
</blockquote>
<h2 id="catalog">Catalog<a class="anchor" aria-label="Link to section" href="#catalog">#</a></h2>
<p><strong>Asset</strong>
A piece of data or something that produces/consumes data. Has a <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>type</span></span></code></span> of <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>dataset</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>report</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>process</span></span></code></span>, or <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>system</span></span></code></span>. Carries <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>:Asset</span></span></code></span> plus a type-specific label in Neo4j.</p>
<p><strong>Actor</strong>
A person or a team. Has a <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>type</span></span></code></span> of <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>person</span></span></code></span> or <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>group</span></span></code></span>. Carries <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>:Actor</span></span></code></span> + <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>:Person</span></span></code></span> or <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>:Group</span></span></code></span>.</p>
<p><strong>Container</strong>
A schema container — typically a table, sheet, or section. Created automatically by <a href="#schema-projection">schema projection</a>; not directly editable.</p>
<p><strong>Field</strong>
A schema column or measure. Lives under a Container. Created by schema projection.</p>
<p><strong>Rule</strong>
A governance / data-quality rule. Has <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>severity</span></span></code></span> (<span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>info</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>warning</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>critical</span></span></code></span>) and a <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>category</span></span></code></span>. Attaches to assets via the <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>applies_to</span></span></code></span> relation, optionally scoped to a <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>field_path</span></span></code></span>.</p>
<p><strong>Event</strong>
An immutable audit record of one mutation (<span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>asset.created</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>relation.deleted</span></span></code></span>, ...). Drives outbound webhooks.</p>
<p><strong>Webhook</strong>
An outbound subscription: URL + event topics + HMAC secret. The API POSTs signed payloads when matching events fire.</p>
<h2 id="relationships">Relationships<a class="anchor" aria-label="Link to section" href="#relationships">#</a></h2>
<p><strong><span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>:OWNS</span></span></code></span></strong> — <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>Actor → Asset</span></span></code></span>. Who is responsible.
<strong><span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>:USES</span></span></code></span></strong> — <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>Actor | Asset → Asset</span></span></code></span>. Who consumes.
<strong><span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>:FEEDS</span></span></code></span></strong> — <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>Asset → Asset</span></span></code></span>. <strong>Lineage</strong> edge — source flows to target.
<strong><span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>:CONTAINS</span></span></code></span></strong> — <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>Asset | Container → Container | Field</span></span></code></span>. Schema tree membership.
<strong><span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>:MEMBER_OF</span></span></code></span></strong> — <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>Person → Group</span></span></code></span>. Team membership.
<strong><span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>:APPLIES_TO</span></span></code></span></strong> — <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>Rule → Asset</span></span></code></span>. Governance attachment, with <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>enforcement</span></span></code></span> and optional <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>field_path</span></span></code></span>.</p>
<h2 id="search">Search<a class="anchor" aria-label="Link to section" href="#search">#</a></h2>
<p><strong>Hybrid search</strong>
The combination of vector (semantic) and fulltext (keyword) ranking, fused into one score per result.</p>
<p><strong>Embedding</strong>
A 384-dim float vector representing an entity's <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>name + description</span></span></code></span>. Generated by BAAI/bge-small-en-v1.5 via fastembed (CPU).</p>
<p><strong>Query DSL</strong>
The small grammar that lets you write <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>ds:</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>owner:</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>feeds:</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>"phrase"</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>-exclude</span></span></code></span> instead of plain text. See <a href="/docs/search">search.md</a>.</p>
<p><strong>Kind</strong>
The bucket a search hit belongs to: <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>asset</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>container</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>field</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>actor</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>rule</span></span></code></span>. Returned in every search hit.</p>
<p><strong>Schema projection</strong>
The process of materialising an Asset's <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>metadata.schema</span></span></code></span> JSON into real <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>:Container</span></span></code></span> and <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>:Field</span></span></code></span> nodes so they're searchable individually. Rebuilt on every asset write.</p>
<h2 id="plugins">Plugins<a class="anchor" aria-label="Link to section" href="#plugins">#</a></h2>
<p><strong>Plugin</strong>
A Python package exposing <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>manifest</span></span></code></span> + <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>async run()</span></span></code></span>, registered via the <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>holocron.plugins</span></span></code></span> entry-point group. Discovered by the API at startup.</p>
<p><strong>Capability</strong>
<span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>IMPORT</span></span></code></span> (returns <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>SummaryResult</span></span></code></span>) or <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>EXPORT</span></span></code></span> (returns <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>DownloadResult</span></span></code></span>).</p>
<p><strong><span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>PluginContext</span></span></code></span></strong>
The services injected into <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>run()</span></span></code></span> — <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>asset_service</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>actor_service</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>relation_service</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>rule_service</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>event_service</span></span></code></span>. Lets the plugin read/write the catalog without going through HTTP.</p>
<p><strong><span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>SummaryResult</span></span></code></span></strong>
The return type for IMPORT plugins: title, counts, samples, optional <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>extra</span></span></code></span> dict.</p>
<p><strong><span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>DownloadResult</span></span></code></span></strong>
The return type for EXPORT plugins: filename, content type, body bytes.</p>
<p><strong>Connector</strong>
A plugin that pulls metadata from an external source (CSV, Excel, Postgres, Power BI, ...) and pushes it into the catalog.</p>
<p><strong>Exporter</strong>
A plugin that produces a file from the catalog (Excel workbook, Markdown zip, ...).</p>
<p><strong>Extension (audit)</strong>
A plugin in the audit / analysis category — <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>lineage-gap-audit</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>compliance-report</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>pii-detector</span></span></code></span>. (Not to be confused with UI extensions.)</p>
<p><strong>UI extension</strong>
A piece of code that contributes commands to the ⌘K palette. Lives under <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>packages/ui/src/extensions/</span></span></code></span>. See <a href="/docs/extensions">extensions.md</a>.</p>
<p><strong><span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>holocron-plugin</span></span></code></span> CLI</strong>
The command-line driver bundled with the SDK. Lists / inspects / invokes plugins on a running API.</p>
<h2 id="identity--audit">Identity &#x26; audit<a class="anchor" aria-label="Link to section" href="#identity--audit">#</a></h2>
<p><strong>UID</strong>
The stable identifier assigned to every entity at creation. Strings like <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>ast-...</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>act-...</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>rul-...</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>evt-...</span></span></code></span>, <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>wh-...</span></span></code></span>. Connectors typically derive UIDs deterministically from source identity so re-scans upsert.</p>
<p><strong><span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>verified</span></span></code></span></strong>
Boolean flag. <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>true</span></span></code></span> means a human has confirmed; <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>false</span></span></code></span> means a plugin guessed it. Connectors create things with <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>verified: false</span></span></code></span> and <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>discovered_by: &#x3C;slug>@&#x3C;version></span></span></code></span>.</p>
<p><strong><span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>discovered_by</span></span></code></span></strong>
String tag identifying the plugin (and version) that created the entity, so you can audit provenance.</p>
<h2 id="operational">Operational<a class="anchor" aria-label="Link to section" href="#operational">#</a></h2>
<p><strong>LOD (Level of Detail)</strong>
Controls how much of the graph the map endpoint returns. <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>lod=0</span></span></code></span> is overview-only (systems + teams), <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>lod=1</span></span></code></span> is everything.</p>
<p><strong>Topic</strong>
The <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>&#x3C;entity>.&#x3C;action></span></span></code></span> string used in webhook subscriptions and <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>X-Holocron-Topic</span></span></code></span>. E.g. <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>asset.updated</span></span></code></span>.</p>
<p><strong>Auto-disable</strong>
After 10 consecutive delivery failures, a webhook is automatically disabled. Re-enable by <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>PUT</span></span></code></span>-ing <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>{"disabled": false}</span></span></code></span>.</p>
