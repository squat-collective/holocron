Plugin catalog

9 plugins, one contract

Connectors push, exporters pull, audits report. All discovered at API startup via the holocron.plugins entry-point group, all driven by the same manifest + async run() contract from the SDK.

CSV connector

CSV/TSV files in

Sniffs encoding, delimiter, and column types. Parses # Owner: comment headers as Person actors. One Dataset per file with schema projected as Container/Field nodes.

Excel connector

.xlsx files in

System / Sheet / Table hierarchy. Detects formal ListObjects + heuristic tables. Captures formula lineage (VLOOKUP / XLOOKUP / INDEX-MATCH), cross-file refs, and owners from custom properties.

Postgres connector

PostgreSQL schemas in

One dataset per table or view, with column-level schema. Idempotent UIDs (host + db + schema + table) so re-runs upsert. Passwords scrubbed from any returned errors.

Walks the Layout JSON and DAX query structure to extract referenced tables and columns. Emits report → table USES edges per visual.

One workbook with Overview, Assets, Actors, Relations, Schemas, Lineage tabs. Round-trips back through the Excel connector for offline review workflows.

Markdown data dictionary

One README.md plus per-asset and per-actor pages. Browses cleanly on GitHub, drops into a docs site, feeds LLMs as context.

Excel workbook flagging orphan assets, lineage dead-ends, undocumented entities, dangling rules, unverified items. Use it as a worklist for catalog hygiene.

Field-name pattern classifier: high-confidence (email, ssn, dob, …) vs medium-confidence (name, country, customer_id). Read-only — reviewers apply flags via Edit field → toggle PII.

Drive plugins from the terminal

The SDK ships holocron-plugin — list, inspect, and invoke any plugin against a running API.

holocron-plugin list --api http://localhost:8100
holocron-plugin show data-dictionary-markdown
holocron-plugin run csv-connector --input file=@orders.csv -o scan.json
