14:T7b45,<h1 id="webhooks">Webhooks<a class="anchor" aria-label="Link to section" href="#webhooks">#</a></h1>
<blockquote>
<p>Outbound events with HMAC-SHA256 signing, fire-and-forget dispatch, and auto-disable on failure.</p>
</blockquote>
<p>Every mutation in Holocron logs an <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>:Event</span></span></code></span> node. Webhooks are subscribers: register a URL plus a list of event topics and you'll receive a signed POST whenever a matching event fires.</p>
<p>See <a href="/docs/architecture/adr/007-outbound-webhooks">ADR-007</a> for the design choice.</p>

<h2 id="verifying-the-signature">Verifying the signature<a class="anchor" aria-label="Link to section" href="#verifying-the-signature">#</a></h2>

<h2 id="delivery-semantics">Delivery semantics<a class="anchor" aria-label="Link to section" href="#delivery-semantics">#</a></h2>
<ul>
<li><strong>Fire-and-forget.</strong> Dispatch happens in an <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>asyncio</span></span></code></span> background task; the API request that triggered the event returns as soon as the event is logged.</li>
<li><strong>Bounded concurrency.</strong> A process-wide <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>asyncio.Semaphore(100)</span></span></code></span> caps in-flight deliveries to prevent runaway tasks.</li>
<li><strong>No persistent retry queue (v0.1).</strong> A failure increments <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>failure_count</span></span></code></span> and stores <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>last_error</span></span></code></span> (truncated to 500 chars) but the event is <strong>not</strong> retried later. v0.2 may add a queue.</li>
<li><strong>Auto-disable.</strong> After 10 consecutive failures the webhook is disabled. Re-enable by <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>PUT</span></span></code></span>-ing <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>{"disabled": false}</span></span></code></span> (which also resets the counter).</li>
<li><strong>HTTP timeout.</strong> 10 seconds per delivery. A timeout counts as a failure.</li>
<li><strong>Receiver responsibilities.</strong> Respond <span data-rehype-pretty-code-figure=""><code data-language="plaintext" data-theme="github-dark-dimmed github-light"><span data-line=""><span>2xx</span></span></code></span> quickly. Idempotency is on you — duplicates are unlikely but not guaranteed (e.g. the API restarts mid-dispatch).</li>
</ul>
